## 一、项目背景

| 项目   | 内容                                          |
| ---- | ------------------------------------------- |
| 产品名称 | 学习打卡追踪器                                     |

| 难度   | ★★                                          |
| 适合角色 | 想做"有成就感"产品的团队                               |
| 涉及技术 | 日期处理、聚合统计、排行榜逻辑、大模型API调用与结构化解析              |

## 三、系统集成需求

系统需设计独立的 **"AI任务解析与分发服务"** ，该服务接收用户输入的文本/语音内容，调用大模型API返回结构化的任务数组，并通过内部接口直接写入打卡任务表，完成从"学习原材料"到"打卡任务"的闭环。**严禁采用"AI只给建议、用户手动复制粘贴再建任务"的割裂体验设计。**

AI服务须支持返回以下标准化字段，以便前端直接渲染和批量创建：

> - `task_name`（任务名称）
> - `estimated_days`（预估天数）
> - `suggested_order`（建议学习顺序）
> - `review_intervals`（复习间隔建议，用于US-20）
> - `knowledge_tags`（知识点标签，用于进度追踪）

## 四、核心差异化亮点

相比普通打卡工具，本项目的差异化不在于"记录得更漂亮"，而在于把"打卡"和"复习"真正打通。

### 4.1 从记录行为，升级为辅助学习

普通打卡产品主要记录"做没做"，本产品进一步辅助"学什么、何时复习、学到哪一步"。

### 4.2 从手动规划，升级为 AI 辅助规划（产品闭环）

用户不需要自行拆分知识点、安排复习节奏。更关键的是——**AI的规划结果直接转化为软件内的打卡任务**，用户只需"确认"，无需"复制粘贴+手动建任务"，真正实现"一句话输入，全套计划生成"。

### 4.3 从独立工具，升级为学习伴侣

基于遗忘曲线的自动复习任务插入，让打卡从"单向记录"升级为"智能复习引擎"，用户不需要额外使用记忆软件，在打卡过程中自然完成复习安排。

### 4.4 从回顾过去，升级为展望未来（日历"前挡风玻璃"）

普通打卡日历只展示"昨天做没做"，本产品将日历升级为"前挡风玻璃"——展示"明天该做什么"。

- **今天格子**：显示"今日任务：xxx"，让用户一眼知道今天该做什么
- **明天格子**：AI动态推荐"预计明日任务：xxx（难度较高，建议预留40分钟）"，帮助用户提前规划时间
- **休息建议**：系统主动推荐休息日（如连续打卡7天后），显示"系统建议你在这天安排休息"，让用户主动安排休息而不是被动断签

本质变化：用户打开日历，看到的是一张**未来的作战地图**，而不是一张过去的成绩单。这会让用户产生"我要往前走"的冲动，而不是"我过去好厉害/好差"的情绪波动。

### 4.5 从比"打卡久"，升级为比"更规律"（排行榜重构）

普通打卡排行榜按"连续天数"排名，鼓励的是"熬时长"，而非"好习惯"。本产品将排行榜重构为**规律性排行**：

- **排序依据**：按"每天同一时段打卡的稳定度"排序，计算用户打卡时间的标准差，标准差越小排名越靠前
- **排名展示**：只展示排名区间（前20%/前30%-50%/后50%），不展示具体排名数字，减少攀比压力
- **奖励机制**：规律性进入前20%获得"规律达人"徽章，前50%获得"习惯养成者"徽章，持续进步获得"进步之星"徽章

本质变化：排行榜从"比谁更能熬"变为"比谁更自律"，引导用户关注"建立稳定的学习习惯"而非"追求天数的堆砌"。
